# 测试用例生成器 (Testcase Generator)

一个帮助 AI 助手自动生成规范、全面测试用例的 skill。

## 功能特性

- **多种输入源支持**：需求文档、禅道需求链接、功能描述、API 接口
- **完整测试类型覆盖**：功能测试、边界测试、异常测试、性能测试、安全测试、兼容性测试
- **多种输出格式**：Excel（默认）、Word、禅道导入 CSV
- **规范化用例编写**：遵循唯一性、可复现、完整性、可断言、规范性原则

## 用例编写原则

| 原则 | 说明 |
|------|------|
| **唯一性** | 一个用例只验证一个核心场景/一个点 |
| **可复现** | 步骤清晰、环境明确、无随机不可控因素 |
| **完整性** | 覆盖正常、异常、边界、空值、错误格式 |
| **可断言** | 所有结果必须可量化、可校验，不能模糊 |
| **规范性** | 字段统一、命名统一、格式统一 |

## 测试用例标准字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 用例 ID | ✅ | 唯一标识，格式：`模块_功能_序号` |
| 用例名称 | ✅ | 格式：`模块 + 场景 + 预期` |
| 所属模块 | ✅ | 归属的功能模块 |
| 优先级 | ✅ | P0(阻塞级)/P1(高级)/P2(中级)/P3(低级) |
| 前置条件 | ✅ | 执行前必须满足的环境/状态 |
| 输入数据 | ✅ | 用户输入、入参字段、关键数据 |
| 测试步骤 | ✅ | 清晰可复现的操作步骤 |
| 预期结果 | ✅ | 可观察、可验证、可量化的结果 |
| 备注 | ❌ | 补充说明、依赖说明 |

## 必须覆盖的场景（5大类）

1. **正常场景**：正确输入 → 正确输出
2. **边界场景**：极限值、临界条件
3. **异常场景**：非预期输入、错误数据
4. **错误处理**：服务超时、无权限、资源不存在
5. **兼容性/格式**：不同环境、格式下的表现

## 安装使用

### ClawHub 安装

```bash
clawhub install testcase-generator
```

### 手动安装

将本仓库克隆到你的 skills 目录：

```bash
git clone https://github.com/Lai-May/testcase-generator.git
```

### 使用示例

```
帮我生成用户注册功能的测试用例，账号11位手机号，密码6-20位含字母数字

根据这个 API 接口生成测试用例：
POST /api/orders
参数：productId(必填)、quantity(必填1-99)、address(必填)

这个支付功能的测试点有哪些？
```

## 输出格式

### Excel 格式（默认）

包含三个 Sheet：
- **测试用例列表**：全部用例明细（含优先级颜色标识）
- **用例统计**：按优先级、类型统计
- **测试覆盖分析**：功能点覆盖情况

### Word 格式

结构化文档，包含：
1. 测试范围概述
2. 测试策略说明
3. 测试用例明细（表格形式）
4. 测试覆盖分析
5. 风险提示

### 禅道导入格式

生成符合禅道导入要求的 CSV 文件。

## 优先级定义

| 级别 | 定义 | 适用场景 |
|------|------|---------|
| P0 | 阻塞级：核心主流程，不修复无法继续测试 | 登录功能崩溃、支付失败 |
| P1 | 高级：重要功能，影响用户核心场景 | 密码修改失败、订单无法查看 |
| P2 | 中级：次要功能，仅影响体验 | 提示文案不规范、样式轻微错乱 |
| P3 | 低级：极边缘场景，几乎不发生 | 连续点击100次无异常 |

## 禁止编写的用例

- ❌ 无明确预期结果（如"看看是否正常"）
- ❌ 一个用例测多个功能
- ❌ 不可复现、依赖随机数据
- ❌ 无断言、无法自动校验
- ❌ 步骤模糊不清
