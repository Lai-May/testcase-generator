# 🧪 测试用例生成器 (Testcase Generator)

[![License: MIT-0](https://img.shields.io/badge/License-MIT--0-yellow.svg)](https://opensource.org/licenses/MIT-0)
[![GitHub stars](https://img.shields.io/github/stars/Lai-May/testcase-generator.svg?style=social)](https://github.com/Lai-May/testcase-generator/stargazers)

> **专业测试用例生成技能，让 AI 助手帮你写出规范、全面、高质量的测试用例！**

---

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🎯 **多种输入源** | 支持需求文档、禅道需求链接、功能描述、API接口文档 |
| 📋 **完整场景覆盖** | 自动覆盖正常、边界、异常、错误处理、兼容性5大类场景 |
| 📊 **多格式输出** | Excel（默认）、Word、禅道导入CSV |
| ⚡ **遵循规范** | 唯一性、可复现、完整性、可断言、规范性五大原则 |
| 🔌 **专项模板** | 内置API测试、安全测试、性能测试、表单测试、CRUD测试模板 |
| 💡 **测试技巧** | 包含等价类、边界值、因果图等测试设计方法 |

## 🚀 快速开始

### 基本用法

直接告诉 AI 你要测试什么功能：

```
帮我生成用户注册功能的测试用例：
- 账号为11位手机号
- 密码6-20位，含字母和数字
- 需要短信验证码（6位数字）
```

### API 测试

```
根据这个 API 接口生成测试用例：
POST /api/orders
参数：productId(必填)、quantity(必填1-99)、address(必填)
```

### 从禅道需求生成

```
根据这个禅道需求生成测试用例：
https://your-zentao.com/story-view-123.html
```

## 📋 测试用例标准字段

| 字段 | 必填 | 说明 | 示例 |
|------|:----:|------|------|
| 用例 ID | ✅ | 唯一标识 | `Login_Account_001` |
| 用例名称 | ✅ | 功能+场景+预期 | `登录 - 密码为空 - 提示必填` |
| 所属模块 | ✅ | 功能模块 | `登录模块` |
| 优先级 | ✅ | P0/P1/P2/P3 | `P0` |
| 前置条件 | ✅ | 执行前状态 | `账号已注册` |
| 输入数据 | ✅ | 测试数据 | `{username: "13800138000"}` |
| 测试步骤 | ✅ | 操作步骤 | 编号列表 |
| 预期结果 | ✅ | 可验证结果 | 具体输出描述 |
| 备注 | ❌ | 补充说明 | 关联信息 |

## 🎯 必须覆盖的5大类场景

```
┌─────────────────────────────────────────────────────────┐
│                    测试场景覆盖矩阵                       │
├─────────────────────────────────────────────────────────┤
│  正常场景 ✅  │ 正确输入 → 正确输出                       │
│              │ 主流程、核心功能验证                       │
├─────────────────────────────────────────────────────────┤
│  边界场景 🔶  │ 极限值、临界条件                         │
│              │ 最大/最小值、边界±1                       │
├─────────────────────────────────────────────────────────┤
│  异常场景 ❌  │ 非预期输入、错误数据                     │
│              │ 空值、格式错误、逻辑错误                  │
├─────────────────────────────────────────────────────────┤
│  错误处理 ⚠️  │ 系统层面异常                            │
│              │ 权限、网络、服务异常                      │
├─────────────────────────────────────────────────────────┤
│  兼容性 🔄   │ 不同环境、格式                           │
│              │ 浏览器、编码、分辨率                      │
└─────────────────────────────────────────────────────────┘
```

## 📊 优先级定义

| 级别 | 名称 | 定义 | 典型场景 |
|:----:|------|------|---------|
| **P0** | 🔴 阻塞级 | 核心主流程，不修复无法继续测试 | 登录崩溃、支付失败 |
| **P1** | 🟠 高级 | 重要功能，影响用户核心场景 | 订单无法查看、密码修改失败 |
| **P2** | 🔵 中级 | 次要功能，仅影响体验 | 提示不规范、样式轻微错乱 |
| **P3** | ⚪ 低级 | 极边缘场景，几乎不发生 | 连续点击100次无异常 |

## 📚 专项测试模板

### 🔌 API 接口测试

自动生成 API 测试用例，覆盖：
- 参数校验（必填、类型、范围）
- 业务逻辑（正常、异常）
- 权限控制（未登录、越权）
- 错误码验证（400/401/403/404/500）

### 🔐 安全测试

内置安全测试场景：
- SQL 注入
- XSS 攻击
- 越权访问
- 敏感数据泄露
- CSRF 攻击
- 暴力破解

### ⚡ 性能测试

生成性能测试用例：
- 响应时间
- 并发测试
- 吞吐量
- 资源占用

### 📝 表单测试

针对表单字段的专项测试：
- 文本框（空值、长度、格式）
- 数字框（范围、小数）
- 下拉框（选项、联动）
- 日期框（格式、范围）
- 文件上传（类型、大小）

### 📊 CRUD 测试

增删改查操作的完整覆盖：
- Create（创建）
- Read（读取）
- Update（更新）
- Delete（删除）

## 💡 示例输出

### 输入

```
登录功能，支持账号密码登录
- 账号为手机号（11位数字）
- 密码6-20位，含字母和数字
- 登录失败3次锁定账号30分钟
```

### 输出（Excel）

| 用例ID | 用例名称 | 模块 | 优先级 | 输入数据 | 预期结果 |
|--------|---------|------|:----:|---------|---------|
| Login_Account_001 | 登录-账号密码正确-登录成功 | 登录 | P0 | {username:"13800138000", password:"Test@123"} | 跳转首页，显示用户名 |
| Login_Account_002 | 登录-密码为空-提示必填 | 登录 | P0 | {password:""} | 提示"密码不能为空" |
| Login_BND_001 | 登录-密码6位-登录成功 | 登录 | P1 | {password:"Abc123"} | 登录成功 |
| Login_EXCP_001 | 登录-手机号不存在-提示错误 | 登录 | P1 | {username:"19999999999"} | 提示"账号不存在" |
| Login_ERR_001 | 登录-账号锁定中-提示锁定 | 登录 | P0 | 账号已锁定 | 提示"账号已锁定30分钟" |
| Login_SEC_001 | 登录-SQL注入-拦截成功 | 登录 | P0 | {username:"admin' OR '1'='1"} | 拦截，提示格式错误 |
